About

Have you ever taken a screenshot of a webpage and used graphics program to mark some important things on it? Or sent your friend a link and instructed him to "Go to the bottom of the page", or "Check third paragraph in the left column" . Wasn't it cumbersome and inefficient? jNottery is here to help users with sharing and managing content of your website more effctively! The library's workflow is fairly straightforward. Whenever you (or your client) add a note to any supported part of a website it will be encoded and saved directly to the URL. Afterwards, anyone can deal with this link as with any other, except that after visiting it, all your notes will show up.

To give it a try click on an image on this webpage, or select some part of the text and use +tt button in the upper-left corner of the current text block. There appear a tooltip with textarea which allows you to add a note.

For more developer specific info check Quick start section and the project's readme file.

When use it?
  • when you maintaining website with a lot of text and images, which users comment on
  • when you're looking for effective way to encourage users to bookmark and share link to your page
  • in early stages of website development when you need to exchange a lot of information about the look and structure of your webapp with clients
  • it's great for any type of content sharing: scores in online games, markers on maps, values of forms etc.

Quick start

It's rather simple to get started with jNottery. If you know jQuery you will feel like at home, otherwise it shouldn't take more than few minutes to figure out how to use it. The main function tt() is a generic wrapper for jQuery() and also a namespace for the whole library. It takes the same arguments as jQuery function (e.g a string indicating css selector or a jQuery object), as well as library's internal types such as tt.core.Note or special jQuery object returned from tt().selection(). Here I will show you how to enable ElementNotes for all images on a website, in 3 easy steps:

1. Include latest jQuery and jNottery files downloaded as an archive from this page. There is nothing special about it, except the doctype declaration which is required by some browsers for proper calculation of window's boundaries. 

<!doctype html>
<!-- ... -->
<head>
<link rel="stylesheet" href="path/to/unzipped/jnottery-v0.1.0-pa.css">
<!-- ... some more html here -->
<script src="https://code.jquery.com/jquery-1.11.0.min.js"></script>
<script src="path/to/unzipped/jnottery-full-v0.1.0-pa.min.js"></script>
</body>

2. Now it's time to initialize the library. Pass the root of your document to the tt function and run .init(). It's rather common to use simply document.body here. The onElementNote callback is called for each element note found in the URL, when document.ready event is propagated. The init function may optionaly handle initialization of tt.vendor submodule, but only when you specify proper Bitly credentials. Please get the access token and add it to your code. It will allow you to test content sharing options. To give user an idea about which elements has notes assigned, we will add garish border to all of them. 

function highlight(note) {
    // note is an instance of tt.core.ElementNote
    tt(note).css('border', '2px solid red'); 
}

$(function() {
    // when document.ready was fired..

    tt(document.body).init({
        vendor: {
            bitly: {
                // your bitly token goes here
                access_token: 'your_bitly_token'
            }
        },   
        onElementNote: highlight 
	});
});

3. The last step is to decide when the tooltip will be shown up. Let's do it simply after clicking on the target image. The library's tooltip works in two ways. When it finds any notes assigned to the element it opens them for preview, or edit. In other case the tooltip's input will be empty and ready for your new content. The new.note.tt event means that new note has been just added. We will highlight its target element as well to keep our interface more consistent. 

$(function() {
    $('img').click(function() {
        var tooltip = tt(this).tooltip({
            // position could be 'top', 'left', 'right' or 'bottom'
            position: 'auto', 
            view: {
                // title of the tooltip
                title: 'Add note to image'
            }
        });

        tooltip.on(
            // event fired when user adds a new note
            'new.note.tt': function(e, tooltip_obj, note) {
                highlight(note);
            }
        );
    });
});

You're ready to merge all the code presented above into a working app. Of course before playing with it make sure you've added some images to your page! Check my sample demo here.
